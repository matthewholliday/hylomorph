Build a small **stdlib-only Python module** that orchestrations import to record every `parent → subagent` call to disk. It is a library, not a CLI. It must drop cleanly into a project scaffolded by the `orchestration-creator` agent (see `orchestrations/<name>/run_orchestration.py`) and be safe to call from a `concurrent.futures.ThreadPoolExecutor`.

## Run directory

On orchestration start, create exactly one run directory:

- Path: `logs/orchestration-<YYYY-MM-DD-HH-MM-SS>/` using **UTC** wall-clock time at start.
- Uniqueness: if that path already exists, append `-2`, `-3`, … until the `mkdir` succeeds. Never overwrite or merge into an existing run directory.
- All other paths in this spec are relative to this run directory.

The orchestration owns this directory for its lifetime and reuses the same handle for every call it logs.

## Filesystem layout

For a call from `<parent>` to `<child>`, append one record to:

```
<parent_chain>/<child>/call.log
```

where `<parent_chain>` is the slash-joined sequence of sanitized agent names from the orchestration root down to `<parent>`. Use the literal string `orchestration` as the synthetic root segment (the orchestration process itself is the root parent).

Examples (run root omitted):

- Orchestration calls `planner` → `orchestration/planner/call.log`
- `planner` later calls `executor` → `orchestration/planner/executor/call.log`
- `executor` calls `summarizer` → `orchestration/planner/executor/summarizer/call.log`

A node's directory may contain **both** a `call.log` (the call into that node) **and** subdirectories for calls that node makes — these coexist; do not move or rename either.

### Name sanitization

Map each agent identifier to a path-safe segment with a single, deterministic rule:

- Replace any character outside `[A-Za-z0-9._-]` with `_`.
- Collapse runs of `_` to a single `_`.
- Strip leading/trailing `.`, `_`, `-`.
- Truncate to 64 chars; if truncation occurs, append `-` plus the first 8 hex chars of `sha1(original)` to keep collisions disambiguated.
- Reject empty results (raise `ValueError`).

## `call.log` format

JSON Lines: one UTF-8 JSON object per line, terminated by `\n`. Required keys per line:

| key | type | notes |
| --- | --- | --- |
| `ts` | string | UTC ISO 8601, e.g. `2026-04-25T20:46:13.421Z`; timestamp of call **completion** |
| `parent` | string | original (un-sanitized) parent identifier |
| `child` | string | original (un-sanitized) child identifier |
| `duration_s` | number | wall-clock seconds, float |
| `result` | object | `{ "status": "success" \| "failure" \| "cancelled" \| "unknown", "error"?: string }` — `error` is a short summary (≤ 500 chars), present only on non-success |
| `extras` | object | optional free-form caller-provided metadata; omit the key entirely if empty |

Example line:

```json
{"ts":"2026-04-25T20:46:13.421Z","parent":"orchestration","child":"planner","duration_s":1.732,"result":{"status":"success"},"extras":{"model":"gpt-5","session_id":"abc123"}}
```

Writes must be a single `os.write` of the encoded line (open with `O_APPEND`, or use a `threading.Lock` per file path) so that interleaved threads cannot corrupt a record. Flush after each write; do not buffer across calls.

## Public API

Expose at minimum:

```python
from dataclasses import dataclass
from pathlib import Path
from typing import Any, Literal

Status = Literal["success", "failure", "cancelled", "unknown"]

@dataclass(frozen=True)
class CallResult:
    status: Status
    error: str | None = None

class RunLog:
    @classmethod
    def create(cls, logs_root: Path | str = "logs") -> "RunLog": ...
    @property
    def run_dir(self) -> Path: ...
    def log_call(
        self,
        *,
        parent_chain: list[str],   # ["orchestration", "planner", ...] — ancestors of the call, root first
        child: str,
        duration_s: float,
        result: CallResult,
        extras: dict[str, Any] | None = None,
    ) -> None: ...
```

- `RunLog.create` performs the unique `mkdir` described above and returns a handle that is safe to share across threads.
- `log_call` creates any missing directories (`mkdir(parents=True, exist_ok=True)`), then appends one record. It must never raise on already-existing directories.
- `log_call` raises `ValueError` for an empty `parent_chain`, an empty `child`, or names that sanitize to empty.
- Logging failures (disk full, permissions) must surface as exceptions to the caller; do not silently swallow.

## Concurrency

The library is used inside orchestrations that fan out with `ThreadPoolExecutor`. Guarantee:

- One `RunLog` instance is safe for concurrent use from multiple threads.
- Records appended concurrently to the same `call.log` end up as whole, separate lines (no interleaving inside a line).

Single-process only. Cross-process coordination (e.g., logging from inside a spawned `cursor-agent` subprocess) is **out of scope**; the orchestration is responsible for logging each `cursor-agent` invocation it makes after `subprocess.run` returns.

## Acceptance criteria

- `RunLog.create()` produces a fresh directory under `logs/`; running it twice in the same second yields two distinct directories.
- The path layout for a 3-deep chain matches `orchestration/<a>/<b>/<c>/call.log`, with `call.log` files at every intermediate node that received a call.
- A `call.log` line round-trips through `json.loads` and contains every required key with the documented types.
- Concurrent `log_call` invocations from 32 threads to the same path produce 32 well-formed lines (verified by tests; see `run_tests.py` conventions used by `orchestration-creator`).
- The module imports nothing outside the Python standard library.
